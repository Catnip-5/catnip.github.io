OOM常见异常

发表于 2019-07-29 更新于 2022-04-19 分类于 JVM 阅读次数：

堆内存溢出

原因

  • java堆内存不够
  • 业务高峰，创建对象过多
  • 内存泄露
  • 内存碎片严重，无法分配给大对象

解决

  • 调整 -Xmx 参数增加堆大小
  • 修复应用程序中的内存泄漏

栈内存溢出

原因

  • 方法调用层次过多（比如存在无限递归调用）
  • 线程栈太小

解决

  • 优化程序设计，减少方法调用层次
  • 调整 -Xss 参数增加线程栈大小

方法区溢出

原因

  • 从 Java 8 开始 Perm gen 改成了 MetaSpace，在本机内存中分配 Class 元数据（称为 MetaSpace）。当MetaSpace 耗尽，则抛出异常

解决

  • 调整 -XX: MaxMetaSpaceSize 参数增加MetaSpace大小
  • 取消 -XX: MaxMetaspaceSize
  • 减小 Java 堆大小,为 MetaSpace 提供更多的可用空间
  • 为服务器分配更多的内存
  • 可能是应用程序 bug，修复 bug

GC开销超过限制

原因

  • GC回收时间过长：超过98%的时间在进行垃圾回收，且回收了不到2%的堆空间，连续多次

解决

  • 调整 -Xmx 参数增加堆大小
  • 使用 -XX:-UseGCOverheadLimit 取消 GC 开销限制
  • 修复应用程序中的内存泄漏

线程创建限制

原因

  • 进程和操作系统的内存资源限制
  • 进程和操作系统的线程数限制
  • Linux中的线程被实现为轻量级进程，因此，还受到pid数量的限制

解决

  • 排查应用是否创建了过多的线程
  • 调整操作系统线程数阈值
  • 增加机器内存
  • 减小堆内存
  • 减少进程数
  • 减小线程栈大小
  • 增大pid范围
